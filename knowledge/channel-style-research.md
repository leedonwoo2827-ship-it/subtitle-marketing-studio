# Channel Style Research — Text Marketing LM

자막(Raw Data)을 17개 텍스트 마케팅 자산으로 재가공할 때 **채널별 톤·길이·포맷·해시태그·CTA 규칙**을 정의한 살아있는 가이드. 각 스튜디오는 자기 `channel_section` 키로 해당 섹션만 발췌해 프롬프트에 주입한다.

---

## PART 0. 사용 철학

- **원칙 1**: 자막 원문은 외부에 노출하지 않는다. 오직 정보·맥락·메시지의 원천(Raw Data)으로만 사용한다.
- **원칙 2**: 채널 톤이 우선. 같은 자막이라도 블로그·뉴스레터·링크드인은 톤·길이·구조가 전혀 다르다.
- **원칙 3**: 검색·발견 가능성(SEO)은 첫 200자에 승부가 난다.
- **업데이트 마커**: 🔎 조사 필요 / ✅ 검증됨 / ⚠ 변동성 높음
- **적합도 표기**: ★★★ 핵심 / ★★ 보조 / ★ 부적합

## PART 2. 채널별 상세 스펙

### 블로그 (네이버 / 티스토리 / Medium)
- 📐 길이: **2,000~4,000자**, H2/H3 소제목 필수, 단락 3~5문장
- 🎭 톤: 정보성·존댓말 기본. **첫 200자에 결론 노출**(검색 결과 스니펫에 잡힘)
- 🪝 후킹: 숫자형(`5가지 이유`) > 질문형 > 단언형 > 시간형(`3분이면`)
- 💬 CTA: 댓글·구독·하트 > 외부 링크. 본문 중간 1회, 말미 1회.
- #️⃣ 해시태그: 본문 하단 5~10개, 한국어·영어 혼합
- 👥 타깃: 25~45세, 정보 탐색·문제 해결 의도
- 📆 발행: 주 2~3회, 평일 오전 9~10시 또는 21~22시
- ⚠ 네이버 검색 알고리즘은 C-Rank/D.I.A. 비중 변동 → 키워드 반복 ≠ 노출
- 🔎 2026년 AI 검색(생성형 답변) 인용 친화 포맷(Q&A 블록·표) 비중 ↑

### 뉴스레터 (스티비 / 메일리 / Substack)
- 📐 길이: **1,200~2,500자**, 인사·본론·맺음·서명 4부 구성
- 🎭 톤: 1인칭 `저는` 또는 `우리는`. 친근하지만 격식 유지. **친구에게 보내는 편지**.
- 🪝 후킹: 제목 30자 이내, 프리뷰 텍스트 90자(받은편지함 미리보기)
- 💬 CTA: 답장 유도 1회 + 외부 링크 1회. **답장**이 가장 강한 시그널.
- #️⃣ 해시태그: 사용 안 함
- 👥 타깃: 자발적 구독자(고관여), 직장인·전문가
- 📆 발행: 주 1회 정기 발송 시간 고정 (예: 화요일 오전 7시)

### LinkedIn
- 📐 길이: **800~1,500자**, 첫 3줄(=`...더 보기` 컷) 안에 후킹
- 🎭 톤: 격식 있는 1인칭. **회사 자랑이 아닌 배운 점(Lesson Learned)**.
- 🪝 후킹: "지난주 한 가지를 배웠습니다." / 숫자(`연봉 10% 인상`) / 역설적 단언
- 💬 CTA: "당신은 어떻게 하시나요?" 질문 1개로 종료
- #️⃣ 해시태그: 3~5개, 본문 끝. `#리더십 #B2B마케팅` 등 영문·한글 혼용
- 👥 타깃: 25~50세 직장인·임원·창업가
- 📆 발행: 화·수·목 오전 8~9시 또는 오후 5시
- ⚠ 외부 링크는 알고리즘이 누른다 → 댓글에 링크 권장

### 인스타그램 카드뉴스 (텍스트 only)
- 📐 1080×1350 [4:5], **5~10장**, 장당 **40자 이내** (한 줄 핵심 문구)
- 🎭 톤: 친근·반말 가능. 1인칭·2인칭(`너`/`당신`).
- 🪝 후킹: **1장(표지)** = 호기심·숫자·반전. 2~4장 = 본론. **5장(또는 마지막)** = CTA·요약.
- 💬 CTA: `저장하기 ↗` / `친구 태그` / `프로필 링크` 중 1개
- #️⃣ 해시태그: 캡션 끝 **10~20개**, 대·중·소 키워드 믹스
- 👥 타깃: 20~35세, 빠른 소비
- 📆 발행: 평일 19~21시, 주말 11~13시
- 🔎 캐러셀 도달이 단일 이미지 대비 ↑ (2025년 기준)

### 커뮤니티 (네이버 카페 / 밴드 / 디시 / 클리앙)
- 📐 길이: **500~1,500자**, 줄바꿈 빈번 (모바일 가독성)
- 🎭 톤: **광고 티 0%**. 정보 공유·경험 나눔 어조. 존댓말 또는 평어.
- 🪝 후킹: `~해봤습니다` / `~정리해드림` / `~경험 공유`
- 💬 CTA: 직접 CTA 금지. 댓글로 질문 유도.
- #️⃣ 해시태그: 사용 안 함 (운영진 차단 사유)
- 👥 타깃: 커뮤니티별 상이. 관심사 일치도가 핵심.
- ⚠ 광고로 인식되면 영구 차단. **브랜드명 노출 최소화**.

### 메타·구글 광고 (피드·검색)
- 📐 후킹 헤드라인 **3종** × 25~40자, 본문 **2종** × 90~125자
- 🎭 톤: 즉각적·이득 중심. 동사로 시작(`알아보세요`, `시작하세요`).
- 🪝 후킹: 숫자·가격·기간(`30일 무료`), 부정 후킹(`이거 모르면 손해`)
- 💬 CTA 버튼: `더 알아보기` / `지금 시작` / `무료 체험`
- #️⃣ 해시태그: 사용 안 함
- 👥 타깃: 타겟팅 모수로 정의 (콘텐츠로 정의 X)
- ⚠ 의료·금융·정치는 정책 거부 사유

### 푸시 알림 / 카카오 알림톡
- 📐 푸시: **30자 이내** 본문. 알림톡: 제목 50자 + 본문 1,000자 한도.
- 🎭 톤: 호기심 유발. 결과 노출 ×, 단서만 ○.
- 🪝 후킹: 의문형(`지금 확인 안 하면?`), 개인화(`OO님, 오늘만`)
- 💬 CTA: 1개 버튼. 클릭 후 랜딩 명확.
- ⚠ 오픈율 핵심 지표. 스팸 단어(`무료`, `긴급`) 남발 → 차단·필터
- 🔎 카카오 알림톡은 사전 템플릿 승인 필요(`#{변수}` 사용)

### 보도자료
- 📐 길이: **800~1,500자**, 5단 구성: 헤드라인 / 부제 / 리드(누가·무엇을·언제·어디서·왜) / 본문 / 회사 소개·연락처
- 🎭 톤: 3인칭 객관. 형용사·과장 금지. **수치·인용** 중심.
- 🪝 후킹: 헤드라인 = `[OO기업] OOO 출시·달성·발표`
- 💬 CTA: 사용 안 함. 연락처·자료 링크만.
- #️⃣ 해시태그: 사용 안 함
- 👥 타깃: 기자·에디터
- 📆 발행: 화·수·목 오전 10~11시 (오후 마감 전)

## PART 3. 발행 플레이북

### 자막 1개 → 채널 확장 (8주 리듬)

| 주차 | 채널 | 스튜디오 |
|---|---|---|
| W1-2 | 블로그 단일 | 01 블로그 |
| W3-4 | + 뉴스레터 크로스 | 02 뉴스레터, 05 SEO 메타 |
| W5-6 | + 인스타·LinkedIn | 03 LinkedIn, 08 인스타 카드 |
| W7-8 | + 광고·푸시 (전환 단계) | 09 PAS, 10 광고, 11 푸시, 12 랜딩 |

### 재사용 규칙
- 자막 1개 → **채널 최대 3개** 동시 운영 권장 (커뮤니케이션 분산 방지)
- 같은 메시지의 채널 간 발행 **최소 7일 간격**
- 인스타 → 블로그(요약 → 확장) ✅ / 블로그 → 인스타(축약) ✅ / 보도자료 → 다른 채널 ❌ (톤 충돌)

- 검증 안 된 항목은 🔎로 표시. 발행 후 실측치(오픈율·도달·클릭)로 ✅ 승격.
- 스튜디오와 채널이 1:1이 아닐 때는 `channel_section`을 다중 발췌하도록 베이스에서 분기.

## 부록 — FAQ

**Q. 자막 원문을 인용하면 안 되나요?**
A. 안 됩니다. 자막은 Raw Data이며, 17개 산출물 모두 재가공된 표현만 노출합니다.

**Q. 한 자막으로 17개를 다 발행해야 하나요?**
A. 아닙니다. 우선 블로그·뉴스레터 2개로 시작하고, 반응을 보며 확장하는 것을 권장합니다(PART 3).

**Q. 채널 가이드와 스튜디오 프롬프트가 충돌하면?**
A. 채널 가이드가 우선입니다. 프롬프트는 자막을 처리하는 *방법*, 채널 가이드는 출력의 *형태*를 정의합니다.
